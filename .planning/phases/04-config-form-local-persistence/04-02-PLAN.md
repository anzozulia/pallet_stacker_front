## Phase Goal

**As a** self-hosting user describing a packing job, **I want to** be warned/blocked when a box can never fit the pallet and see a live running total of what I am asking the solver to do, **so that** I catch typos before running and understand the size of the job.

## Artifacts this phase produces

- **`src/lib/box-fit.ts`:** `orientationsFor(box)` (enumerates allowed `[footA, footB, up]` orientations per rotation mode) and `checkAllBoxesFit(config)` returning `{ ok, failures: FitFailure[] }` where `FitFailure = { index, id, message }`. Conservative: rejects only genuinely-impossible boxes (D-01).

- Co-located `*.test.ts` golden suites for both.

<feature>
  <name>Box-fits-pallet conservative feasibility check (D-01 / BOX-06)</name>
  <files>src/lib/box-fit.ts, src/lib/box-fit.test.ts</files>
  <behavior>
    Cases (golden, from RESEARCH §"Box-fit feasibility check" design note):
    - exact-fit: a box whose footprint exactly matches pallet L×W and whose height ≤ pallet height → fits (ok, no failure).
    - overhang-edge: a footprint slightly larger than pallet L×W but within `pallet.maxOverhang` → fits; beyond overhang → fails.
    - too-tall-but-rotatable: a box too tall for `pallet.height` in its base orientation but with `rotation: 'free'` it can lay a shorter axis up → fits; the SAME box with `rotation: 'fixed'` → fails (height axis cannot change).
    - uprightOnly: height axis fixed up; base may turn 90° (footprintFits already tries both axis alignments) → one orientation entry.
    - 2000mm typo: a 2000mm box on an 800mm pallet (40mm overhang, 1800mm height) → fails in every mode.
    - `checkAllBoxesFit` aggregates per-box failures with `{ index, id, message }`; `ok` is true iff `failures` is empty.
    - exhaustiveness: the `switch (rotation)` ends with a `never` default so adding a RotationMode is a compile error.
  </behavior>
  <implementation>
    Implement per RESEARCH §"Box-fit feasibility check (D-01) — box-fit.ts (pure)". `orientationsFor`: `fixed` → `[{footA:L,footB:W,up:H}]`; `uprightOnly` → `[{footA:L,footB:W,up:H}]` (height fixed; the footprint helper covers the 90° turn); `free` → three entries (each axis up). `footprintFits(a,b,pL,pW,overhang)` returns true if `(a<=pL+overhang && b<=pW+overhang) || (a<=pW+overhang && b<=pL+overhang)`. A box fits if `orientationsFor(b).some(o => o.up <= pallet.height && footprintFits(o.footA,o.footB,pallet.length,pallet.width,pallet.maxOverhang))`. Failure message: `` `"${b.label}" cannot fit the pallet in any allowed orientation` `` (matches UI-SPEC copy). Type-only imports from `@/types/config`. House-style header (purpose + "imports nothing runtime: no three/React/IO"). Conservative principle: when in doubt, allow — only reject the impossible.
  </implementation>
</feature>

Type-only import of BoxType. House-style header.
  </implementation>
