## Phase Goal

**As a** self-hosting user describing a packing job, **I want to** fill a validated form, see a live running total, be blocked on impossible input, run to get my request, and have everything survive a refresh, **so that** I can hand a correct request to the packer with zero signup and nothing lost.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user input → form state → localStorage | Form values (incl. box labels) are user-entered, validated by the resolver, persisted to localStorage |
| localStorage → app (on mount) | The restored blob is untrusted; hydrated via the pure guard (Plan 03) |
