    Cases (golden, from RESEARCH Pattern 7 + Pitfall 4):
    - round-trip: `deserializeConfigOrDefault(serializeConfig(DEFAULT_CONFIG))` deep-equals DEFAULT_CONFIG.
    - null: `deserializeConfigOrDefault(null)` → DEFAULT_CONFIG.
    - unparseable: `deserializeConfigOrDefault('not json{')` → DEFAULT_CONFIG (no throw).
    - version mismatch: a blob `{ version: 99, config: <valid> }` → DEFAULT_CONFIG.
    - shape fail: a blob `{ version: 1, config: { pallet: 'oops' } }` (wrong types / missing keys) → DEFAULT_CONFIG.
    - lenient draft restores: a blob `{ version: 1, config: <DEFAULT_CONFIG with one box length set to 0> }` (structurally valid, business-invalid) → restores that config (NOT defaults), proving the restore guard uses the lenient shape schema, satisfying D-04 "persistence captures work-in-progress even when invalid".
    - never throws: every malformed input returns a PackConfig, never raises.
  </behavior>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localStorage → app (on hydration) | The raw stored blob is untrusted: a user (or another script) may have hand-edited or corrupted it. It crosses into the app via `deserializeConfigOrDefault`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-4-PERSIST | Tampering / DoS | deserializeConfigOrDefault on malformed/oversized localStorage | mitigate | `JSON.parse` wrapped in try/catch; version guard + `safeParse` against an explicit schema → silent fallback to DEFAULT_CONFIG. Never throws, never crash-loads (D-07). Pinned by the unparseable/version/shape-fail tests. |
| T-4-PP | Tampering | prototype pollution via parsed object | mitigate | The parsed object is validated by `safeParse` against an explicit zod schema (unknown keys discarded); the parsed blob is never spread into a shared/global object before validation. |
| T-4-04 | Information Disclosure | sensitive data in localStorage | accept | Stored data is non-sensitive pallet/box dimensions only — no PII, no credentials (security_context). No retention/compliance control needed. |
</threat_model>
