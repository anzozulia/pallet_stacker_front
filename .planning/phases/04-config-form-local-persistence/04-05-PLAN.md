## Phase Goal

**As a** self-hosting user describing a packing job, **I want to** add, edit, and remove box types with all their dimensions, weight, quantity, max-load, fragile flag, and rotation mode, and see a live count, **so that** I can describe my full box catalog.

## Artifacts this phase produces

- **`src/features/config/BoxRow.tsx`:** props `{ index: number; control; register; allIds: string[]; onRemove: () => void }` (or equivalent RHF-context wiring). Renders the colour swatch (13×13, via `colorForType(allIds)` looked up by this row's `id`), the editable name (`label`) input, the read-only id-tag, the dims (L/W/H mm via `NumberField`), weight/unit (kg), quantity (pcs), max-load (kg), the `Fragile` `Switch`, the `Allowed rotation` `SegmentedControl` (3 modes), and a trash remove button with `aria-label="Remove {label}"`. Implements the fragile↔maxLoad interaction (RESEARCH Pattern 4).
- **`src/features/config/BoxCatalogCard.tsx`:** uses `useFieldArray({ name: 'boxTypes' })`; maps rows keyed on `field.id` (NOT index); `Add box type` appends `makeDefaultBoxType()` then scrolls+focuses the new row; a live `{N} types · {M} units` badge via `useWatch` + `tallyCatalog`; the empty-catalog state ("No box types yet" / "Add at least one box type to run the packer.") when the array is empty.
