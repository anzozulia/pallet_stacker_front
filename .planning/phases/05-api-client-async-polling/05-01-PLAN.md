## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser client → packing API (`POST /pack`, `GET /jobs/{id}`) | The first real network IO in the app. The fetch response is untrusted JSON crossing into the render tree. |
| build-time env → runtime client | `VITE_API_URL` is baked at build; a misconfigured value resolves to an unreachable origin (handled, not crash). |
| npm registry → build | New supply-chain surface: two packages added this wave. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-5-01 | Tampering / DoS | `GET /jobs/{id}` response parsed in `client.ts` | mitigate | zod `jobStateSchema` at the boundary (C-02 / ASVS V5) — a malformed/changed body is a handled ZodError (→ contract-drift), never an `as`-cast that crashes the render. Non-strict objects tolerate forward-compat fields (Pitfall 5) so a benign API minor-bump does not false-positive. |
| T-5-02 | DoS (availability) | opaque cross-origin fetch failure | mitigate | `classifyFetchError` buckets any `TypeError` (opaque CORS — verified: API sends no ACAO for a cross-origin browser call) as `unreachable`; the client NEVER reads `res.status` off a thrown fetch (Pitfall 2). |
| T-5-03 | (info) abort mis-surfaced as error | cancel/unmount aborting in-flight fetch | mitigate | `classifyFetchError` returns `aborted` for `DOMException 'AbortError'` (Pitfall 3), keeping cancel a UI no-op (consumed by the Wave-3 LoadingPage). |
| T-5-SC | Tampering | npm installs (`@tanstack/react-query`, `msw`) | mitigate | Task 1 supply-chain checkpoint (T-5-SC, blocking-human): both verified on npmjs.com, mature/high-download, pinned exact; msw `postinstall` reviewed (in-package worker copy, no external network). slopcheck was unavailable → human gate is mandatory and non-auto-approvable. |
