it deterministically (never the live API, per CLAUDE.md).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| API error body (`error.message`/`problems`) → error card render | Untrusted server-supplied strings rendered into the DOM. |
| transport failure (POST/poll throw) → terminal-state classification | An opaque thrown fetch must be bucketed, never read for a status. |
| cancel/unmount → in-flight request lifecycle | Leaving the route must abort cleanly with no zombie request/interval. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-5-10 | XSS (Tampering) | ErrorCard rendering `error.message`/`problems` | mitigate | Rendered as React text (auto-escaped); NEVER `dangerouslySetInnerHTML` (ASVS V5). The error body is also zod-parsed (Wave 1 `errorBodySchema`) before reaching the card. |
| T-5-11 | DoS (availability) | the four terminal states + transport throws | mitigate | Every state has a handled branch (failed/timeout/unreachable cards, done→/result); a thrown POST/poll is `classifyFetchError`-bucketed (Pattern 3); `unpacked>0` is treated as SUCCESS (not a crash/error, Anti-Pattern). None reach an unhandled render. |
| T-5-12 | DoS (resource exhaustion) | cancel / browser-Back / unmount | mitigate | `AbortSignal` on POST + poll + react-query auto-cancel on disable/unmount (SC-3 / PACK-05); `AbortError` classified as `aborted` (no error card flash, Pitfall 3); e2e + unit assert no leaked request/interval. |
| T-5-13 | Tampering | code-split leak via the new ErrorCard / loading feature dir | mitigate | `src/features/loading/*` + `/loading` stay three-free; `scripts/check-code-split.mjs` (Task 2) is the build gate (C-06). |
</threat_model>
