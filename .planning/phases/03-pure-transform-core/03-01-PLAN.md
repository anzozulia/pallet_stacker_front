<objective>
Establish the pure, zero-runtime-import type foundation that both Phase 3 transforms consume and that Phase 4's form will fill. Declare the canonical in-memory app config model (D-01) and the POST /api/v1/pack request contract, and consolidate the existing done-response interfaces into `src/types/` (D-02) behind a re-export shim so no existing importer changes.

This plan creates the shared interface seam (interface-first ordering): Plan 02 (request-builder) and Plan 03 (result-mapper) both import from these type modules and can then run in parallel.

Purpose: Phase 3 is inside-out ‚Äî the pure layer OWNS the form‚Üîbuilder and client‚Üîmapper contracts. Defining the types first means the two transform plans build against fixed contracts instead of inventing shapes.
Output: `src/types/config.ts` (app model), `src/types/pack-contract.ts` (request + response contracts), `src/lib/fixture-types.ts` reduced to a re-export shim. No logic, no tests in this plan ‚Äî types only.
</objective>
