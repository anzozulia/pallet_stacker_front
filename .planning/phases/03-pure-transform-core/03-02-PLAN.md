## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03 | Tampering / Integrity | item-id generation in `buildPackRequest` | mitigate | ID-collision or non-determinism would break type recovery (a data-integrity defect, not a web threat). Mitigated by the deterministic `${typeId}-${index}` counter + the explicit "ids are unique" and "deterministic across rebuilds" tests. Random generators (nanoid/uuid) are forbidden by acceptance criteria. |
| T-03-04 | Tampering | rotation mode → API string mapping | accept | An out-of-domain rotation value cannot occur: `RotationMode` is a closed union and `Record<RotationMode, ApiRotation>` is compile-time-total (tsc -b in build). No runtime branch can emit an invalid `rotations` string. Rationale: the type system is the control. |
| T-03-05 | Information Disclosure | pure module import surface | accept | No IO, no network, no secrets, no DOM. The builder produces an in-memory object; nothing is transmitted in this phase (API client is Phase 5). The grep-0 purity criterion is the backstop. |
| T-03-SC | Tampering | npm/pip/cargo installs | mitigate | Zero packages installed this phase (RESEARCH § Package Legitimacy Audit). `nanoid` is available but deliberately unused and explicitly forbidden by acceptance criteria. No supply-chain surface. |

This is a pure, offline, IO-free, network-free transform. No `high`-severity threats. The realistic risk is ID non-determinism (data integrity), which is test-pinned.
</threat_model>
