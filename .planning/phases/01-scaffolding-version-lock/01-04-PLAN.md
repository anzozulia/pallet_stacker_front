<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Docker build context → image | Source + build args enter the image; secrets/history must be excluded |
| internet → nginx container (port 8080) | Public requests reach the static file server; only the SPA shell + hashed assets are served |
| CI runner → repo / registry | CI installs deps and runs builds; lockfile must be authoritative, no secrets leaked |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-08 | Information Disclosure | Docker build context | mitigate | .dockerignore excludes .git, *.md, coverage, e2e; VITE_API_URL is a public base URL (not a secret), so baking it is safe |
| T-1-09 | Elevation of Privilege | nginx container running as root | mitigate | nginxinc/nginx-unprivileged:alpine runs as UID 101 on port 8080; no USER root, no port 80 (D-14) |
| T-1-10 | Information Disclosure | deep-link fallback exposing files | mitigate | try_files /index.html only serves the SPA shell + hashed /assets/; root scoped to dist; no directory listing |
| T-1-11 | Tampering | CI dependency resolution | mitigate | CI uses npm ci (not install) against the committed lockfile; no secrets in workflow; VITE_API_URL public |
| T-1-SC | Tampering | npm package installs in CI/image | mitigate | npm ci enforces the registry-verified, exact-pinned lockfile from Plan 01 (RESEARCH.md Package Legitimacy Audit) |
</threat_model>
