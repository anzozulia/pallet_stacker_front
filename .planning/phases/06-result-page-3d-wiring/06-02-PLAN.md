| react-query cache → render | The cached `done` body (already zod-parsed at the Phase-5 network boundary) is read and rendered. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-06-02 | Tampering / Information Disclosure | API-sourced strings (item_id, pallet_id) rendered into the topbar/scene chrome | mitigate | Render as React text children only — never `dangerouslySetInnerHTML`/`innerHTML` (mirrors Phase-5 ErrorCard escaped-text rule, ASVS V5). Verified by `grep -c dangerouslySetInnerHTML == 0`. |
| T-06-03 | Tampering | Crafted `history.state` jobId/idToType | mitigate | `isResultNavState` validates `jobId` is a string + `idToType instanceof Map` before use; `getQueryData(['job', jobId])` returns undefined for an unknown id → redirect to `/`. Mirrors `isLoadingNavState`. |
| T-06-04 | Denial of Service | Malformed/oversized cached `done` crashing the render | mitigate | The redirect-on-absent guard + the `done.status === 'done'` check bound the shape before `mapDoneResponse`; the body was zod-parsed upstream (Phase 5). Wrong-shape `done` fails the status guard → redirect, not crash. |
