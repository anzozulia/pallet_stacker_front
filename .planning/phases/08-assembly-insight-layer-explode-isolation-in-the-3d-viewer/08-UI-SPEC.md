# Phase 8 — UI Design Contract

> Visual and interaction contract for the Assembly Insight viewer layer (Explode slider + Layer-focus control). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Brownfield phase.** This phase EXTENDS the existing `/result` 3D viewer (Phases 2/6). It introduces no new page, no new design system, and no new token group. It adds one bottom-center control bar to `ViewerOverlay.tsx` and a row-click interaction to `PlacementList.tsx`. The contract below is therefore mostly a re-statement of the already-established system the new controls MUST match exactly, plus the net-new interaction specs the controls introduce.

## Spacing Scale

Declared values (all multiples of 4 — Tailwind utility scale; spacing is NOT tokenised in `@theme`, components use utilities directly):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-less inline gaps, slider tick spacing, chip vertical padding |
| sm | 8px | Compact pill/button gaps (existing toggles use `gap-2` = 8px) |
| md | 16px | Default control-bar inner gaps, label↔slider spacing |
| lg | 24px | Overlay edge insets (existing chrome anchors at `left-6`/`bottom-6` = 24px) |
| xl | 32px | Layout gaps between control groups |
| 2xl | 48px | Major section breaks (not expected in this phase) |
| 3xl | 64px | Page-level spacing (not expected in this phase) |

**Exceptions (carry-forward from existing viewer chrome — match these, do not "normalize"):**
- Overlay pill buttons use `px-[10px] py-[6px]` (10px / 6px) — the established control padding in `ViewerOverlay.tsx`. The new control bar's buttons/toggles MUST use the SAME `px-[10px] py-[6px]`.
- Overlay edge inset is `24px` (`*-6`); the new bottom-center bar anchors at `bottom-6` (24px from the bottom edge), horizontally centered with `left-1/2 -translate-x-1/2` (mirrors the existing top-center toggle cluster).
- Bottom-center bar must clear the bottom-left control hints (`drag/scroll/right-drag`) and the bottom-right presets — it is centered between them on the same 24px baseline; if widths collide on narrow viewports it may sit slightly above the 24px line (desktop-first, D-03).

## Typography

The viewer overlay is mono-first (it is technical chrome over a 3D scene). The new controls live in this overlay and MUST follow the existing overlay type contract — not the light config-form contract.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Control label (slider name, e.g. "Explode", "Layers") | 12px (`text-xs`) | 400 (regular) for the word, 600 (semibold) only on the active/emphasis token | 1 (`leading-tight` / `leading-none` as used) |
| Control value readout (e.g. "Layer 3 / 5", "0.6×") | 12px (`text-xs`), mono, `tabular-nums` | 400 | 1 |
| Button / toggle text (Build-up / Isolate, etc.) | 12px (`text-xs`), mono | 400 | `leading-tight` |
| Title / dims tag (existing, unchanged) | 16px title (`text-base`, 600) / 12px tag (`text-xs`, 400) | as existing | `leading-tight` |

Constraints: **3 effective sizes in this phase** (16px existing title, 12px chrome, 9.5px field micro-label reused only if a layer label appears in `PlacementList`). **2 weights**: regular (400) + semibold (600). All overlay chrome text is `font-mono`. No new size or weight may be introduced.

## Color

This phase paints onto the dark 3D-overlay token group (`--color-d-*`). The accent is reserved and already in use for the "active" state of the existing toggles/presets. The new controls reuse the exact same active/inactive treatment.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--color-d-bg` `#0c0f17` | Scene background behind the overlay; the canvas is the dominant surface |
| Secondary (30%) | `#1a2030` (inactive control fill) + `--color-d-border` `#222838` (borders) + `#1a2030`/white-5% chip fills | Control-bar surfaces, inactive pill/toggle backgrounds, slider track |
| Accent (10%) | `--color-accent` `#4f46e5` (active fill) with `border-[rgba(124,116,255,0.6)]` and white text | RESERVED — see list below |
| Destructive | `--color-danger` `#dc2626` | NOT used in this phase (no destructive action exists here) |

**Accent reserved for (explicit list — never "all interactive elements"):**
1. The **active state** of the Build-up / Isolate mode toggle (the selected mode pill).
2. The **slider fill / thumb** for the Explode and Layer sliders (the filled portion of the track + the draggable thumb), so the active value reads against the inactive `#1a2030` track — mirroring how the existing toggles use accent only when ON.
3. Nothing else. Inactive pills, the slider track groove, labels, value readouts, and the bar background all use the `--color-d-*` / `#1a2030` greyscale group. Hover on inactive controls uses `hover:bg-[#222a3d] hover:text-[var(--color-d-text)]` (the established overlay hover), NOT accent.

**Scene-material colours are NOT UI tokens** (per `src/styles.css` header): the per-type box palette, the ghost/dimmed opacity for isolated layers (D-09), wood, grid, and lights are 3D-material module constants in the viewer, not `@theme` tokens. The ghost opacity value for the Isolate dim is implementer's discretion (CONTEXT D-09 / Claude's Discretion) — a single translucent alpha applied to non-focused layer meshes; recommend ~0.12–0.18 alpha so context is kept without competing with the focused layer.

## Copywriting Contract

This phase has no form, no empty data set, no network call, and no destructive action. The "copy" surface is the new control labels and their value readouts. Honesty-first (L-06): never editorialize beyond what the geometry says.

| Element | Copy |
|---------|------|
| Primary CTA | None — this phase has no submit/CTA. The primary interactions are the **Explode** slider and the **Layers** slider; treat their labels as the load-bearing copy. |
| Explode control label | `Explode` (left-anchored label) with a right value readout. Readout at 0: `Assembled`. Readout when raised: a unitless `0.0×`–`1.0×` (or step value) — implementer picks the scalar but the 0 endpoint MUST read `Assembled`, not `0×`, to make the no-op default legible (SC-3 honesty). |
| Layers control label | `Layers` with a mode toggle reading `Build-up` / `Isolate` and a slider value readout. Default readout: `All` (= build-up-full, all layers shown). When focused: `Layer {k} / {N}` (floor-up, 1-based per D-08). |
| Mode toggle copy | Two pills: `Build-up` and `Isolate`. Default selected: `Build-up`. (D-08/D-09.) |
| Empty / single-layer state | If `computeLayers` returns a single layer (`N === 1`), the Layers slider has no range to traverse. Show the controls **disabled with the readout `Single layer`** (greyed, `pointer-events-none`, reduced opacity) rather than a broken 0…0 slider. Explode is likewise a visual no-op at N=1 — keep it enabled but it produces no gap (harmless, do not special-case beyond the layer count). |
| Error state | Not applicable — `computeLayers` is a pure in-memory derivation over already-validated placements; it cannot fail at runtime. If `selMapped.items` is empty (no boxes on the pallet), render the control bar **disabled** with readout `No boxes`. No error UI. |
| Destructive confirmation | None — no destructive action in this phase. |

**Discoverability copy (D-02 always-visible-at-defaults):** the bar is always rendered. At defaults the scene is byte-identical to the Phase-6 assembled view. The labels (`Explode` / `Layers`) ARE the affordance — no separate "Assembly insight" disclosure button, no tooltip required.

## Net-New Interaction Contract (Phase-8 specific)

These are interaction contracts the checker/auditor should hold the implementation to. They are derived from the locked CONTEXT decisions; restated here so the executor has one visual source of truth.

| Interaction | Contract |
|-------------|----------|
| Bottom-center control bar (D-01) | New absolute-DOM strip, `pointer-events-auto` on the controls only (the wrapping container stays `pointer-events-none` per L-04). Dark-overlay tokens + the SAME pill/border/`px-[10px] py-[6px]` styling as the existing toggles. Anchored `bottom-6 left-1/2 -translate-x-1/2`. |
| Always-visible at no-op defaults (D-02 / SC-3) | Explode = `Assembled` (0), Layers = `All` (build-up-full, all visible), mode = Build-up. Scene unchanged until a control is touched. Not hidden behind a disclosure. |
| Explode slider (D-04/D-07) | Uniform additive gap (layer `i` lifts by `i × slider × fixedUnit`). Transition is **animated** (spring/tween, easing implementer's call). Accent slider fill + thumb. |
| Camera auto-fit while exploding (D-05) | Explode change re-frames the camera to the growing bbox. MUST NOT fire on pallet switch (D-02 preserves camera there). Visual: camera dollies/reframes smoothly as the stack grows. |
| CoG marker hidden while exploded (D-06) | `explode > 0` hides the CoG marker; `explode === 0` re-shows it (respecting the existing CoG toggle ON/OFF state). Pallet deck/wireframe stays at the base. |
| Layers slider + mode toggle (D-08/D-09) | Single slider `0…N`, floor-up. Build-up = show layers `1..k`, upper layers **hidden**. Isolate = show only layer `k`, others **dimmed to translucent ghosts** (kept for context). |
| Compose (D-10) | Explode + Layer-focus apply simultaneously. Isolate + Explode = single layer (explode is a harmless no-op; do NOT disable). |
| Reset on pallet switch (D-11) | `setSel` resets Explode → 0 and Layers → default. Camera still preserved across the switch. |
| Placement row click → isolate (D-12) | Clicking a `PlacementList` card sets mode = Isolate, focus index = that box's layer (revealing it if hidden). Existing one-way hover→mesh emissive (Phase 6 D-11) is unchanged and composes. Clicked/active row SHOULD show a persistent selected cue distinct from the transient hover cue (e.g. accent left-border or ring) so "this row drove the current isolation" is legible. Reuse `accent` / `accent-weak` already in the card's hover styling — do not introduce a new colour. |
| Optional layer label on rows | A small read-only `Layer {k}` chip MAY accompany each placement card (planner discretion, Deferred-Ideas note). If added, reuse the existing micro-label style (`font-mono text-[9.5px] uppercase tracking-[0.08em] text-text-3`) — no new typography. |

## Accessibility

- Mode toggle pills follow the existing pattern: `role="switch"` (or radio-group semantics for the two-way Build-up/Isolate choice) with the visible label as the accessible name and `aria-checked`/`aria-pressed` reflecting state — mirroring the existing CoG/heatmap toggles in `ViewerOverlay.tsx`.
- Sliders use a native `<input type="range">` (or an accessible slider primitive) with `aria-label` (`Explode amount`, `Layer focus`), `aria-valuetext` matching the visible readout (`Assembled`, `Layer 3 / 5`, `All`), and keyboard arrow-key operability.
- All control state is fully readable WITHOUT hover (focus=hover parity, established UI-SPEC rule) — the readouts always show the current value.
- Contrast: accent `#4f46e5` with white text and inactive `#1a2030` with `--color-d-text-2` `#838b9e` are the already-shipped, signed-off overlay combinations; reuse them as-is.
