# Phase 2 — UI Design Contract

> Visual and interaction contract for the static, explorable 3D pallet viewer on the `/result` route.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Visual source of truth: `design/result.html` (viewer pane only) + CONTEXT.md decisions D-07..D-14.

---

## Scope Boundary (read before consuming this contract)

This phase builds **only the 3D viewer hero pane** of the result screen and its **on-canvas chrome**:

**IN (this phase):**

- The 3D scene: wood pallet (slats + blocks), key/fill/ambient/hemisphere lighting, PCF soft shadows, ground plane + grid, radial-gradient background + fog, per-type-coloured boxes with `EdgesGeometry` edge-lines.
- Dark overlay header (top-left): pallet name + dimensions tag. **STATIC labels** sourced from the fixture's pallet 0 dimensions — NO computed sub-line.
- Legend (top-right): swatch + type-name rows.
- Control hints (bottom-left): `drag` orbit · `scroll` zoom · `right-drag` pan. Static.
- Camera preset buttons (bottom-right): ISO / TOP / FRONT, with active-state styling.
- Orbit / zoom / pan interaction + animated preset transitions.

**OUT (explicitly Phase 6 — do NOT build, per D-14 / D-15):**

- The entire right rail: the topbar (brand, step indicator, "Solved in Xs" pill, Edit/Export buttons), summary 2×2 stats block, pallet switcher, placement list, hover↔mesh highlight, CoG marker, support-ratio tinting, unpacked-items panel.
- The dark-overlay computed sub-line (`12 boxes placed · 84.3% fill · 101.0 kg`) — needs the Phase 3 mapper.
- Any fill % / weight / box-count overlay stat.

For this phase the viewer fills the route (the `100dvh` wrapper from the current `ResultPage`); the 384px rail column from the mockup is NOT laid out yet.

### Token additions to `src/styles.css` (D-08 — EXTEND the existing `@theme`, do not duplicate `--font-sans` / `--font-mono` / `--color-accent`)

Add the dark 3D-overlay token group, ported verbatim from `design/result.html` `:root`:

```css
@theme {
  /* existing: --font-sans, --font-mono, --color-accent: #4f46e5 — keep */

  /* dark 3D-overlay group (D-08) */
  --color-d-bg: #0c0f17; /* scene background base / fog colour */
  --color-d-border: #222838; /* overlay chip + button borders */
  --color-d-text: #e6e8ee; /* primary overlay text (titles) */
  --color-d-text-2: #838b9e; /* secondary overlay text (tags, hints, legend) */
}
```

Note: scene-only colours that are NOT design tokens (wood, lights, ground, grid, box-type palette) live as constants in the viewer module, not in `@theme` — they are 3D-material values, not reusable UI tokens.

## Spacing Scale

Declared values (multiples of 4). The overlay chrome uses the small end of the scale; the mockup's odd values (26/30/24px insets) are snapped to the nearest 4-multiple.

| Token | Value | Usage                                                        |
| ----- | ----- | ------------------------------------------------------------ |
| xs    | 4px   | Swatch↔label gap, button icon gaps                           |
| sm    | 8px   | Legend row gap, preset-button gap, button padding-x baseline |
| md    | 16px  | Control-hint inter-item gap, overlay title↔tag gap           |
| lg    | 24px  | Overlay edge insets (top / left / right / bottom)            |

Exceptions:

- Preset buttons and legend swatches use sub-4 internal values that match the mockup exactly and are intentional micro-chrome, not layout spacing: swatch `9×9px`, preset/tag border-radius `5–6px`, button padding `6px 10px`. These are visual-fidelity carve-outs (D-07 "faithful to the mockup") and are NOT subject to the 8-point layout rule.

## Typography

All overlay chrome text. Two weights only (regular 400 + semibold 600). Mono is used for tags, hints, legend, and preset buttons (matching the mockup's `--mono` usage); sans-semibold is the pallet title.

| Role           | Size | Weight                      | Line Height | Font           | Usage                                                                                                                |
| -------------- | ---- | --------------------------- | ----------- | -------------- | -------------------------------------------------------------------------------------------------------------------- |
| Viewer title   | 16px | 600                         | 1.2         | Inter          | Pallet name in the overlay header (mockup 15px → snapped to 16px body anchor)                                        |
| Dimensions tag | 12px | 400                         | 1.2         | JetBrains Mono | `1200 × 800 × 1800 mm` chip beside the title                                                                         |
| Legend label   | 12px | 400                         | 1.2         | JetBrains Mono | Box-type name per legend row (mockup 11px → snapped to 12px)                                                         |
| Control hint   | 12px | 400 (label) / 600 (keyword) | 1.2         | JetBrains Mono | `drag orbit` etc.; the verb (`drag`/`scroll`/`right-drag`) is weight 600 in `--d-text`, the rest 400 in `--d-text-2` |
| Preset button  | 12px | 400                         | 1.2         | JetBrains Mono | ISO / TOP / FRONT labels                                                                                             |

Body baseline (inherited from existing global, for reference): 16px / 400 / 1.5 Inter. The viewer chrome runs tighter (1.2) because it is dense single-line UI, not running prose.

## Color

The viewer is a **dark surface** — its 60/30/10 split is internal to the dark scene, distinct from the app's light shell (which lands in Phase 4/6). Within the viewer pane:

| Role            | Value                                                                                                                | Usage                                                                      |
| --------------- | -------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------- |
| Dominant (60%)  | `#0c0f17` (`--color-d-bg`), as a radial gradient `#161d2b → #0c0f17`                                                 | Scene background + fog. The dark canvas is the dominant surface.           |
| Secondary (30%) | `rgba(255,255,255,0.05)` fills on `#222838` (`--color-d-border`) borders                                             | Overlay chips, preset-button backgrounds — the floating-glass chrome layer |
| Accent (10%)    | `#4f46e5` family — active preset button only: bg `rgba(99,90,245,0.32)`, border `rgba(124,116,255,0.6)`, text `#fff` | The single reserved accent in this pane                                    |
| Destructive     | not applicable                                                                                                       | No destructive actions in this phase                                       |

Accent reserved for: **the active camera-preset button (ISO / TOP / FRONT) only.** Nothing else in the viewer uses the indigo accent. (The app-shell accent uses live elsewhere in later phases; within this viewer, accent === the selected-view highlight.)

### Box-type colour palette (D-09) — NOT a 60/30/10 token; a categorical scale

Deterministic palette seeded by the mockup's three colours, assigned to box types in fixture order. The mapping must be stable (same type → same colour across renders), keyed off the type identifier from the fixture.

| Index | Hex                 | Mockup origin                                                                                                                                                                                                                                                                   |
| ----- | ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 0     | `#6d63f5`           | indigo (mockup `BX-01` "Standard carton")                                                                                                                                                                                                                                       |
| 1     | `#0ea5a3`           | teal (mockup `BX-02` "Heavy tote")                                                                                                                                                                                                                                              |
| 2     | `#e0892b`           | amber (mockup `BX-03` "Glassware tray")                                                                                                                                                                                                                                         |
| 3+    | extend harmoniously | If the fixture has >3 types: generate additional hues by even spacing around the colour wheel from the seed hues (e.g. rose `#e0506b`, violet `#9b5de5`, green `#52a447`), keeping similar saturation/lightness so the legend reads as one family. Deterministic by type index. |

Box material: `MeshStandardMaterial` `roughness 0.62`, `metalness 0.04` (mockup values). Edge-lines: `LineSegments(EdgesGeometry)` tinted from the box colour via `offsetHSL(0, -0.04, +0.18)`, `transparent`, `opacity 0.55` (mockup `tint()`).

### Scene material / lighting constants (D-07 fidelity — module constants, not tokens)

| Element                           | Value                                                                                 |
| --------------------------------- | ------------------------------------------------------------------------------------- |
| Ambient light                     | `#9fb0d0` @ 0.55                                                                      |
| Hemisphere light                  | sky `#cdd7f0` / ground `#0b0d14` @ 0.5                                                |
| Key (directional, shadow-casting) | `#fff6e8` @ 1.15, pos ~(900, 1750, 1150), 2048² shadow map, PCF soft                  |
| Fill (directional)                | `#6d7cff` @ 0.35, pos ~(-1000, 600, -800)                                             |
| Ground plane                      | `#0e1320`, roughness 1, receives shadow                                               |
| Grid                              | lines `#243049` / `#19202f`, opacity 0.5, transparent (drei `<Grid>` or `GridHelper`) |
| Wood — deck/slat top              | `#caa06a`, roughness 0.85                                                             |
| Wood — sides/blocks               | `#b08a55`, roughness 0.85                                                             |
| Fog                               | `#0c0f17`, near ~2800 far ~5600 (tune to fixture scale via auto-fit, not hardcoded)   |
| Renderer                          | antialias on, pixelRatio capped at 2, PCFSoftShadowMap                                |

Note: fog/shadow/camera distances in the mockup are hardcoded to its 1200×800×1800 pallet. Per D-11/D-12 these MUST be derived from the fixture's bounding box (auto-fit), not copied verbatim, so any real fixture frames correctly.

## Interaction Contract (camera & controls — RESULT-02)

| Interaction       | Behaviour                                                                                             |
| ----------------- | ----------------------------------------------------------------------------------------------------- |
| Orbit             | left-drag — drei `<OrbitControls>` (or `<CameraControls>`), damping on (`dampingFactor ≈ 0.08`)       |
| Zoom              | scroll — min/max distance derived from scene bounding-box size (D-12), not hardcoded                  |
| Pan               | right-drag                                                                                            |
| Polar clamp       | `maxPolarAngle ≈ π·0.495` so the camera never orbits beneath the ground plane (D-12)                  |
| ISO preset        | three-quarter elevated view echoing the mockup's ISO composition; auto-fit framed to the fixture bbox |
| TOP preset        | straight-down view (camera directly above target, looking down)                                       |
| FRONT preset      | axis-aligned eye-level view looking along one horizontal axis                                         |
| Preset transition | animated (~520ms, ease-out cubic feel per mockup); active button reflects current preset              |
| Initial frame     | start at ISO so the first painted frame is composed                                                   |

Presets MUST visibly reframe (TOP looks straight down; FRONT looks along an axis) and MUST auto-fit so a real-dimension fixture is always fully composed. Do NOT copy the mockup's hardcoded camera vectors (D-11).

## Copywriting Contract

All copy is **static** this phase (no computed values). Pallet name + dimensions come from the fixture's pallet 0 record.

| Element                  | Copy                                                                                                                                                                              |
| ------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Primary CTA              | none — this phase has no buttons that submit/mutate. The only controls are the three camera-preset toggles.                                                                       |
| Camera preset buttons    | `ISO` · `TOP` · `FRONT` (uppercase, exactly as mockup)                                                                                                                            |
| Viewer title             | Pallet name from fixture — default `Pallet A` if the fixture exposes no human name (pallet index 0).                                                                              |
| Dimensions tag           | `{L} × {W} × {H} mm` from the fixture pallet 0 dimensions, integer millimetres, multiplication sign `×` (U+00D7), single spaces. Example: `1200 × 800 × 1800 mm`.                 |
| Control hints            | `drag` orbit · `scroll` zoom · `right-drag` pan (verb bold in `--d-text`, action word in `--d-text-2`)                                                                            |
| Legend rows              | swatch + the box-type display name from the fixture/type map (fallback to the raw type id if no friendly name exists, e.g. `BX-01`)                                               |
| Empty state              | not applicable — the fixture is committed and always present; the viewer never renders against missing data this phase. (Loading/empty/error states for live data are Phase 5/6.) |
| Error state              | not applicable this phase — fixture is static and bundled; a malformed-fixture failure is a build/test failure surfaced by the golden Vitest tests, not a runtime UI state.       |
| Destructive confirmation | none — no destructive actions in this phase                                                                                                                                       |
